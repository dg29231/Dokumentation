Sprint 1

linked sketch

Das Kunstwerk «Cosmic Songs» von Zdeněk Sýkora visualisiert ein Gedicht von Jan Neruda. Das Kunstwerk hat mich mit seinen schwungvollen Linien sehr angesprochen. Die Arbeit mit dem «Bezier» im Code war etwas anspruchsvoll. In Illustrator kann man den Kontrollpunkt mit der Maus beliebig setzen, im Code muss man eine genaue Zahl eingeben. Mit der Funktion «Random» konnte ich beliebig lange und breite Linien zeichnen. Die S-Form bleibt aber bewusst immer erhalten. Als nächstes habe ich die Farbvariationen angepasst, zwei zufällige Zahlen für die Dicke eingefügt und Punkte in das Bild gesetzt. Die Ergebnisse sind sehr schlicht. Der weisse Hintergrund erzeugt viel Ruhe. Lebendiger wurde das Bild, als mehrere Linien miteinander kombiniert wurden. Das Bild wirkt wild. Durch die reduzierte Deckkraft im Hintergrund kommt wieder mehr Ruhe ins Bild. Die Linien und Punkte in meinem Code erinnern mich an eine Melodie.

Da das Kunstwerk «Cosmic Songs» ein Gedicht visualisiert, haben wir uns entschieden, mit dem «Volume»-Sensor zu arbeiten. Die ersten Versuche waren auch mit dem «Bezier». Die «Bezier»-Linie verzerrt sich, wenn die Lautstärke erhöht wird. Durch die starke Veränderung der Lautstärke hat die Linie keinen Schwung mehr. Sie ist zu starr. Also habe ich einen ganz neuen Code geschrieben, der nicht auf der ersten Ausgabe basiert. Der neue Code ist eher eine Wellenlinie, die höher oder tiefer wird, wenn sich die Lautstärke ändert. Das visuelle Ergebnis, das auf den Lautstärkesensor reagiert, hat mich zu wenig angesprochen. Die Linien erinnern mich zu sehr an eine Lautstärkekurve.

Mir ist es wichtig, dass mein Ergebnis schwungvoll und harmonisch aussieht. Durch den Lautstärke-Sensor springen die Linien zu stark. Sie wirken wild und verwirrend. Das harmonische Zusammenspiel ist nicht mehr zu erkennen. Deshalb habe ich meine statischen, aber schwungvollen Bilder in eine Animation verpackt. Die Geschwindigkeit des Bildwechsels hängt von der Lautstärke im Raum ab. Je lauter es ist, desto schneller wechselt das Muster.

Sprint 2

Das Coden mit HTML und CSS im Sprint 2 war für mich neu. Weil ich keine Basisgrundlage habe, ist es für mich immer noch oft unverständlich, wie der Code genau funktioniert.

Ich habe mich von der Web-Extension von Rafaël Rozendaal «Abstract Browsing» inspirieren lassen. Da ich mir wirklich nicht vorstellen konnte, wie diese Web-Extension funktioniert, habe ich mit ChatGPT einen Code erstellen lassen. Der Code war unglaublich kompliziert geschrieben und habe ihn kaum verstanden. Somit versuchte ich ihn umzuschreiben, um den Code für mich verständlich zu machen. Diese Herausforderung hat leider nicht einwandfrei funktioniert. Da P5 bereits Variablen und Funktionen zuvor definiert hat, musste man vor jeder Funktion oder Variable ein «sketch» schreiben.

Die Web-Extension von Rafaël Rozendall ist mir aber etwas zu bunt. Darum habe ich unterschiedliche Farbpaletten ausgesucht und mich für eher ruhige, helle Farben entschieden. Normalerweise arbeite ich immer mit den RGB-Farben. ChatGPT hat den Code aber bereits im Farbcode vorgeschrieben. Leider hat die Umschreibung auf RGB nicht funktioniert.

Zudem versuchte ich, dass die Web-Seite beim Scrollen unsichtbarer wird. Wenn man ganz unten auf der Webseite ist, sieht man nur noch einen weissen Hintergrund und die Schrift «Take a Break!». Die Herausforderung mit dem Scrollen und dem weissen Hintergrund war bis zum Schluss eine grosse Herausforderung, die ich aber noch meistern konnte.

Grundsätzlich versuche ich immer ohne ChatGPT einen Code zu generieren. Mir ist es wichtig, dass ich den Code selberverstehe und eine logische Abfolge habe. Bei HTML und CSS funktioniert das leider noch nicht. Die Hilfe, welche ich von ChatGPT beansprucht habe, möchte ich in einem weiteren Projekt wieder minimieren und den Code so gut wie möglich selbständig aufgleisen.

In Sprint 2 haben wir als Gruppe beschlossen, den eigenen Projekten nachzugehen. Wir haben uns gegenseitig unterstützt, wenn man beim Code nicht mehr weiterkam, und liessen uns gegenseitig von den unterschiedlichen Projektideen inspirieren.

Sprint 3

linked sketch

Beim Sprint 3 nutzte ich die Möglichkeit mich in diverse Codes einzutauchen. Dank Daniel Shiffman habe ich zwei komplexe Codes kennengelernt und der Aufbau davon gut verstanden. «Worley Noise» war ein Code und der andere «Marching Squares». Da Daniel Shiffman mit «Processing» die Codes geschrieben hat, habe ich seine Erklärungen in meine Code-Kenntnis «übersetzt». Meine Codes sind wahrscheinlich nicht perfekt und besonders schöngeschrieben, aber ich verstehe die Funktion dahinter und den Aufbau. Zudem habe ich sie somit eigenständig geschrieben und nicht «einfach» abgeschrieben. Es sind zwei komplexe Videos von Daniel Shiffman. Ich habe das Gefühl, sehr viel davon gelernt zu haben.

Im Hinterkopf hatte ich immer, dass ich einen Code-Output Sticken möchte. Sticken ist eine beruhigende, erholsame Ablenkung vom Alltag. Das Tempo wird reduziert und die Abläufe wiederholen sich immer wieder. Marching Squares bietet die perfekte Mustergrundlage für ein Stickmuster. In Kombination mit dem Code für die Erstellung eines Pixel-Bildes konnte ich das Stickmuster digital umsetzen. Der Analoge Prozess war eine perfekte Ablenkung vom Bildschirm, dem technischen und komplexen Coden.

Der analoge und digitale Output hat eine ganz andere visuelle Wirkung. Das gestickte Muster bewirkt einen 3D-Effekt, während im digitalen nur 2D gearbeitet werden kann. Zudem können der Stoff und der Faden angefasst und gefühlt werden. Alle die bereits die Handarbeit Sticken ausgeführt haben, wissen, wie aufwendig es ist. Geduld ist gefragt. Bewusst habe ich mich für ein bestimmtes Muster entschieden, dass ich in 2.5 h gestickt habe. Anders ist es beim digitalen Muster im Code. Hat man den Code einmal geschrieben, könnten hunderte von unterschiedlichen Mustern exportiert werden.

In dieser Arbeit möchte ich zeigen, wie wichtig es ist, das Tempo zu reduzieren und sich bewusst auf eine Tätigkeit zu konzentrieren.

Die Präsentation des Endprodukts hätte ich gerne mit einer Beamer Installation gemacht. Während der digitale Output sich regelmässig verändert, bleibt die gestickte Arbeit ruhig hängen. Aus technischer und räumlicher Organisation des Festivals habe ich ein Mockup zur möglichen Installation erstellt.
